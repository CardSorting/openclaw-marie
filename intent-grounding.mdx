Intent Grounding is a mission-critical pre-processing phase that transforms ambiguous human requests into high-fidelity, validated, and contextually aware technical specifications. It moves the agent from **Stochastic Generation** (guessing what to do) to **Deterministic Execution** (working from a verified plan).

## Grounding vs. Planning: A Critical Distinction

It is common to confuse "Grounding" with "Planning." However, they serve two distinct architectural purposes:

- **Planning**: Defines the **sequence of actions** (e.g., "1. Read file, 2. Run command, 3. Edit code"). It is about _how_ to solve the problem.
- **Grounding**: Defines the **reality of the problem** (e.g., "The user says 'Fix the auth' which means 'Modify `src/auth/service.ts` while adhering to the `JWT_EXPIRY` rule'"). It is about _what_ the problem actually is in the context of the filesystem and project rules.

**Grounding is the "Input Filter" for Planning.** Without Grounding, Planning is based on hallucinations.

## The 6-Pass Hardening Protocol

The Intent Grounding strategy has been hardened through a rigorous 6-pass protocol, evolving from a foundational schema to a state-of-the-art mastery system:

1.  **Pass 1: Schema Foundation**: Enforces a strict Zod-backed structure (`GroundedSpec`) for all outputs.
2.  **Pass 2: Environment Discovery**: Real-time extraction of project markers (frameworks, tech stack, documentation).
3.  **Pass 4: Clarification Loop**: Interactive CLI recovery when intent is ambiguous or information is missing.
4.  **Pass 4: Contextual Filtering**: Semantic keyword-based and Markdown-aware filtering of project rules to maintain high relevance.
5.  **Pass 5: Self-Validation**: Similarity-based drift detection and path-sanity verification to detect hallucinations.
6.  **Pass 6: Mastery & Security**: Alignment memory (multi-turn consistency), configurable rule discovery, and security guardrails for high-risk commands.

## Architectural Components

### 1. The Grounding Engine (`IntentGrounder.ts`)

The core orchestrator responsible for:

- **Rule Ingestion**: Reading and filtering configuration files (`.cursorrules`, `README.md`, etc.).
- **Environment Discovery**: Detecting monorepos, frameworks (Next.js, React), and TypeScript status.
- **LLM Orchestration**: Steering the grounding model with standardized XML-tagged prompts.
- **Self-Correction**: Retrying failures with "Advisory Hints" for the LLM.

### 2. The Specification (`GroundedSpec`)

A machine-readable contract containing:

- `intent`: The refined, actionable goal.
- `rules`: specific constraints identified during grounding.
- `confidence`: A probabilistic score (0-1) governing the clarification trigger.
- `metadata`: Comprehensive telemetry (latency breakdown, drift scores, retry counts).

### 3. The Clarification Loop

When confidence drops below the threshold (default: 0.7) or `missingInfo` is detected, the agent pauses and prompts the user via the CLI. The answers are injected into a recursive "Re-grounding" pass to finalize the intent.

## Advanced Production Features

### Security Guardrails

The grounder monitors the sharpened intent for high-risk patterns:

- **Absolute Paths**: Detections of `/private/...` or `C:\...` automatically lower confidence.
- **Destructive Commands**: Presence of `rm -rf`, `sudo`, or `chmod` triggers a security warning and forces low confidence for human review.

### Alignment Memory

To maintain multi-turn consistency, the grounder is seeded with **Alignment Memory** (previous grounded intents from the session history). This ensures that "Step 2" of a task remains grounded in the context established during "Step 1".

### Performance Profiling

Detailed `latencyBreakdown` tracks time spent in:

- `discovery`: Workspace scanning.
- `prompt`: Rule ingestion and prompt construction.
- `llm`: Token generation time.

## Configuration

Grounding behavior is highly configurable via `openclaw.json`:

```json
{
  "agents": {
    "defaults": {
      "groundingModel": "anthropic/claude-3-5-sonnet",
      "groundingRules": [".openclawrules", "DOCS.md"]
    },
    "list": [
      {
        "id": "security-expert",
        "groundingRules": ["SECURITY_POLICY.md"]
      }
    ]
  }
}
```

## Codebase Deep Dive

### Core Logic

- **`src/agents/grounding/IntentGrounder.ts`**: The main entry point.
  - `ground()`: Orchestrates the multi-pass grounding process.
  - `discoverEnvironment()`: Scans the workspace for tech-stack markers.
  - `readRuleFiles()`: Ingests project-specific rules with Markdown-aware filtering.
  - `hardenIntent()`: The LLM-driven refinement loop.
  - `calculateSimilarity()`: Detects semantic drift between original and grounded intents.

### Types & Schemas

- **`src/agents/grounding/types.ts`**:
  - `GroundedSpec`: The Zod-validated output format.
  - `GroundingContext`: The input parameters (workspace, history, agent config).

### Integration Points

- **`src/commands/agent.ts`**: (CLI Entrypoint)
  - Triggers grounding before agent spawning.
  - Implements the **Clarification Loop** via `promptQuestion`.
  - Propagates session history for **Alignment Memory**.
- **`src/agents/model-selection.ts`**:
  - `resolveGroundingModel()`: Routes grounding requests to the dedicated model if configured.
- **`src/config/types.agents.ts`** & **`src/config/types.agent-defaults.ts`**:
  - Adds `groundingModel` and `groundingRules` to the configuration schema.

## Strategic Impact

| Failure Mode            | Grounded Implementation Solution                                                        |
| :---------------------- | :-------------------------------------------------------------------------------------- |
| **Hallucinated Files**  | **Path-Sanity**: Rejects any absolute or non-relative path not found in discovery.      |
| **Ambiguity Paralysis** | **Clarification**: Interactively asks for missing parameters.                           |
| **Rule Violation**      | **Elite Rules**: Fallback constraints ensure relative paths and JoyZoning are followed. |
| **Contextual Drifting** | **Alignment Memory**: Seeds current grounding with previous results.                    |
